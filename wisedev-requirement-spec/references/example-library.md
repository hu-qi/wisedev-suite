# 高质量样例库：需求规格说明书

## 样例 1：政企协同模块需求规格

### 输入前提
已有需求梳理稿，已明确角色、场景、功能范围。

### 推荐文档结构
1. 建设背景与目标
2. 适用对象与角色定义
3. 业务场景
4. 范围内 / 范围外
5. 功能需求（FR 编号）
6. 非功能需求
7. 业务规则
8. 验收标准
9. 风险与待确认事项

### 功能需求条目示例
#### FR-001 调度指令下发
系统应支持巡办管理员向一个或多个巡组下发调度指令，指令应包含主题、正文、截止时限、附件和回执要求。

#### 验收标准示例
- 给定巡办管理员已登录，创建调度指令并选择两个巡组后，系统应向两个巡组产生待办。
- 当巡组签收后，系统应更新该指令的签收状态与签收时间。
- 当超过截止时限未反馈时，系统应显示逾期状态并允许催办。

## 样例 2：短需求扩写时的克制边界

### 原始输入
实现周报填报。

### 推荐扩写方式
- 先补：谁填、填给谁、什么时候填、填什么、如何查看、如何催报
- 不要直接扩写成复杂工作流，除非用户明确提出审批、退回、汇总分析等要求

### 稳定输出片段
#### 角色
- 填报人
- 审阅人
- 汇总查看人

#### 功能
- 新建周报
- 保存草稿
- 提交周报
- 查看历史周报
- 按组汇总查看

## 样例 3：非功能需求写法示例

### 推荐写法
- 安全性：涉密材料下载、预览、流转应受权限控制，并记录操作日志。
- 可用性：常用查询操作的平均响应时间应小于 3 秒。
- 审计性：关键业务动作应记录操作者、时间、对象、结果。

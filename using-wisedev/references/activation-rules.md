# WiseDev 激活规则

本文件用于在任何相关任务开始前，先判断是否需要启用 WiseDev 套件。

## 一、默认应激活 WiseDev 的任务

以下任务默认应激活 WiseDev：
- 需求梳理
- 需求规格说明书
- 概要设计 / 详细设计
- OpenAPI 契约
- Vue + Mock 原型
- 任意两阶段及以上的连续交付

## 二、必须先进入总控 Skill 的情形

以下情形必须先进入 `wisedev-orchestrator`：
1. 输入材料模糊或混合。
2. 用户没有明确当前阶段。
3. 上游产物不足但要求下游产物。
4. 用户一次要求多个阶段的结果。

## 三、允许直接进入子 Skill 的情形

仅当同时满足以下条件时，允许直接进入子 Skill：
- 用户明确要求单阶段产物；
- 输入材料已经满足该阶段最低前提；
- 该请求不要求继续串联下游阶段。

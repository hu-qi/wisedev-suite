# 高质量样例库：会议纪要 / 转写 → 需求梳理

## 样例 1：上传下达模块

### 原始输入
实现上传下达模块，实现巡办与巡组之间的双向信息传递，包括工作调度、专项材料传递、会议协同、组上报等工作。还要考虑巡办与纪检、组织、职能部门的横向沟通与任务协同。

### 稳定输出骨架

#### 一页式摘要
本模块面向巡办、巡视巡察组及协同部门，建设安全、可追踪、可督办的双向协同通道，支撑工作调度、材料传递、会议协同、请示上报与跨部门协作。

#### 角色清单
- 巡办管理员
- 巡组组长
- 巡组成员
- 协同部门联络员
- 系统管理员

#### 核心场景
1. 巡办向巡组下发调度指令
2. 巡办向巡组安全传递专项材料
3. 巡组向巡办上报周报和问题清单
4. 巡组发起协助申请，巡办协调外部部门
5. 巡办对未按时反馈事项发起催办

#### 需要显式补出的待确认问题
- 专项材料的密级等级如何定义
- 是否需要二次签收或阅后即焚能力
- 横向协同部门的权限边界如何划分
- 会议协同是否需要视频会议集成

## 样例 2：督查督办模块

### 原始输入
需要做一个督查督办模块，领导布置事项后，责任单位要分解任务并反馈进度，督办人员能看到逾期情况并催办。

### 稳定输出骨架

#### 功能归并结果
- 督办事项登记
- 责任单位签收
- 子任务拆解与责任到人
- 周期性进度反馈
- 逾期预警与催办
- 统计看板与归档

#### 风险与歧义项
- “反馈进度”是文本反馈还是标准表单
- 是否允许责任单位退回事项
- 逾期判定按自然日还是工作日

## 样例 3：转写内容较乱时的处理方式

### 输入特征
- 存在大量“这个那个”“后面再说”
- 多人插话，角色混乱

### 推荐处理方式
1. 先抽取人、事、对象、动作。
2. 再归并成“谁对谁做什么”。
3. 最后再落成功能条目、约束和待定问题。

### 不推荐做法
- 直接逐段重写纪要
- 把口头原话搬进需求草案
- 对未讨论的审批流和异常流做强行扩写

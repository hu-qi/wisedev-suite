# 跨阶段审阅清单

## 通用检查
- [ ] 是否显式区分“已确认事实”和“工作假设”
- [ ] 是否存在范围蔓延
- [ ] 是否存在术语不一致
- [ ] 是否保留需求 → 设计 → API → 原型的追踪关系

## 文档检查
- [ ] 章节结构是否完整
- [ ] 编号是否连续
- [ ] 用语是否正式、稳重、非互联网口语化

## 契约检查
- [ ] 路径命名是否资源化
- [ ] 响应和错误是否标准化
- [ ] 示例是否足够支持前端 mock

## 原型检查
- [ ] 能否本地运行
- [ ] 路由、页面、mock、状态管理是否齐全
- [ ] 是否能够演示关键业务流程

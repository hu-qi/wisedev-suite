# 高质量样例库：概要设计 / 详细设计

## 样例 1：模块拆分

### 输入前提
需求规格已明确“上传下达模块”包括调度、材料传递、会议协同、组上报、协助申请。

### 稳定模块划分
- 调度指令模块
- 材料传递模块
- 会议协同模块
- 组上报模块
- 协助申请模块
- 督办预警模块
- 统计看板模块

### 输出方式
每个模块至少写清：职责、入口、核心流程、状态、权限、数据对象、异常。

## 样例 2：状态机写法

### 对象
调度指令

### 推荐状态
草稿 → 已发布 → 已签收 → 办理中 → 已反馈 → 已归档

### 异常状态
- 已逾期
- 已撤回
- 已作废

### 设计说明写法
状态迁移必须说明触发动作、执行角色、前置条件、是否允许回退。

## 样例 3：异常处理写法

### 场景
专项材料传递

### 推荐补充项
- 接收方无权限时，返回无权访问并记录审计日志
- 附件病毒扫描失败时，阻断提交并提示重新上传
- 材料已过保密时效时，禁止继续转发

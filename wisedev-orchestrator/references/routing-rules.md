# 路由规则

- 含大量讨论文本、转写、纪要 → `wisedev-meeting-to-spec`
- 用户要求“形成正式需求规格说明书” → `wisedev-requirement-spec`
- 用户要求“概要设计 / 详细设计 / 模块设计” → `wisedev-design-doc`
- 用户要求“接口契约 / OpenAPI / mock-ready API” → `wisedev-openapi-contract`
- 用户要求“可运行原型 / Vue 演示 / mock 页面” → `wisedev-vue-mock-prototype`

如果用户输入同时跨多个阶段，先生成阶段计划，再逐段产出。

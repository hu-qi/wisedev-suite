# 高质量样例库总览

本文件用于帮助总控 Skill 在不同 AI 工具中更稳定地识别“输入起点”“当前目标阶段”“下一个最合适的子 Skill”。

## 使用方式

1. 先将用户输入归类到最接近的样例类型。
2. 优先复用样例中的阶段判断和输出顺序。
3. 如实际输入比样例更完整，可跳过前置阶段；如更模糊，则先补“上下文包”。
4. 不要机械照抄样例内容，要保留字段结构、阶段边界和交接方式。

## 样例 1：简短口头需求 → 先做需求梳理

### 输入特征
- 只有一句话或一小段目标描述
- 包含若干功能点，但缺少角色、边界、异常、约束
- 用户常说“先帮我梳理一下需求”

### 典型输入
实现上传下达模块，实现巡办与巡组之间的双向信息传递，包括工作调度、专项材料传递、会议协同、组上报等工作。

### 阶段判断
- 当前最合适阶段：需求梳理
- 目标子 Skill：`wisedev-meeting-to-spec`

### 推荐输出顺序
1. 上下文包
2. 一页式需求梳理
3. 角色与场景
4. 功能点与边界
5. 待确认问题
6. 建议下一步：正式需求规格说明书

## 样例 2：会议纪要 / 转写 → 先做需求简报

### 输入特征
- 输入较长，包含反复讨论、口语化表达、插话
- 需求点较多，但状态不统一
- 用户常说“根据会议纪要实现需求梳理”

### 阶段判断
- 当前最合适阶段：会议整理与需求简报
- 目标子 Skill：`wisedev-meeting-to-spec`

### 交付重点
- 区分明确结论、倾向性方案、待确认事项
- 去噪，不保留口头语
- 补齐结构，不过度脑补细节

## 样例 3：已有需求规格 → 进入设计阶段

### 输入特征
- 已有较规范的功能列表、角色、验收要求
- 用户要“概要设计”“详细设计”

### 阶段判断
- 当前最合适阶段：设计文档
- 目标子 Skill：`wisedev-design-doc`

### 推荐交接内容
- 背景与目标
- 功能编号
- 角色权限
- 非功能约束
- 需要覆盖的重点流程

## 样例 4：已有规格 / 设计 → 进入 API 契约阶段

### 输入特征
- 用户明确要求 `openapi.yaml`
- 或者要求“先把接口契约定出来，便于 mock”

### 阶段判断
- 当前最合适阶段：API 契约
- 目标子 Skill：`wisedev-openapi-contract`

### 推荐输出顺序
1. 接口范围说明
2. 假设清单
3. `openapi.yaml`
4. Mock 示例
5. 待确认字段或错误码

## 样例 5：已有 API / 设计 → 进入 Vue + Mock 原型阶段

### 输入特征
- 用户明确要“本地可跑的 Vue 原型”
- 希望能演示复杂流程和多角色业务

### 阶段判断
- 当前最合适阶段：前端原型与 mock
- 目标子 Skill：`wisedev-vue-mock-prototype`

### 推荐输出顺序
1. 项目结构
2. 页面清单
3. 路由图
4. mock 场景矩阵
5. 关键代码文件
6. 本地运行说明

# 原型设计方向指南

## 1. 方向设计的目标

本阶段要解决的问题不是“页面怎么画”，而是：
- 这套原型给谁看
- 应该显得更稳健，还是更有看板指挥感
- 哪些视觉选择能帮助业务信息更快被理解

## 2. 流派池与默认三分法

### 流派池 1：秩序信息型 `enterprise_order`
- 适合：正式汇报、政企评审、材料答辩
- 气质：可信、克制、结构化
- 常见风险：容易变成普通后台

### 流派池 2：数据指挥型 `data_command`
- 适合：看板、调度、督办、监管、中台
- 气质：控制台、信号感、行动导向
- 常见风险：容易过密、过强

### 流派池 3：极简精致型 `minimal_refined`
- 适合：高层简报、抽象业务方案、品牌统一要求高的演示
- 气质：秩序、留白、精致
- 常见风险：信息承载能力不足

### 流派池 4：结构实验型 `experimental_system`
- 适合：创新项目、方案比稿、需要视觉记忆点的演示
- 气质：张力、节奏、结构创新
- 常见风险：喧宾夺主

### 流派池 5：东方现代型 `modern_east`
- 适合：高层汇报、方案展示、对外交流
- 气质：克制、留白、精致、带一点文化辨识度
- 常见风险：若处理不好，会牺牲业务效率

## 3. WiseDev 默认三分法

### A. 稳健政企型 `stable_enterprise`
- 适合：立项汇报、评审会、领导演示
- 气质：克制、可信、秩序感强
- 典型做法：浅底色、分层卡片、清晰标题等级、少量强调色
- 风险：过于保守时会显得像普通后台

### B. 数据指挥型 `data_command`
- 适合：看板、调度、督办、监管、中台场景
- 气质：控制台、信号感、行动导向
- 典型做法：更强信息面板、状态色更明确、指标区更有存在感
- 风险：如果强调过度，页面会显得拥挤和压迫

### C. 东方现代型 `modern_east`
- 适合：高层汇报、方案展示、对外交流
- 气质：克制、留白、精致、带一点文化辨识度
- 典型做法：温和中性色、局部朱砂/墨蓝强调、留白更大
- 风险：若处理不好，会牺牲业务效率

默认情况下，这 3 套可分别对应：
- 安全解：`stable_enterprise`
- 主推解：`data_command`
- 差异化解：`modern_east`

如任务需要，也可以改成：
- `stable_enterprise + minimal_refined + modern_east`
- `stable_enterprise + data_command + experimental_system`

实际输出时，尽量保证几套方案之间有明显反差。

## 4. 迭代建议

- 可以直接选定一套方向深化
- 也可以在必要时混合两套方案中的部分元素
- 若做混合，建议保留一个主骨架方向，避免下游 theme package 失稳

## 5. 页面级落地原则

### Dashboard
- 必须先看到“结论”再看到“细节”
- Hero 只讲一件事，不堆满文案
- KPI 卡不超过 4 到 6 张

### List
- 筛选区优先服务演示任务，而不是模拟复杂后台
- 状态标签要一眼可扫，不靠正文解释
- 行操作遵循主次分明

### Detail
- 顶部先给对象身份，再给当前状态和下一步动作
- 时间轴和说明区应明显分层

### Form
- 优先降低讲解负担，字段成组出现
- 校验信息要具体，不要只有“请输入”

### Workflow
- 强调阶段关系和卡点，不要只做流水账
- 当前节点和异常节点应有明确视觉差异
